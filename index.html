Abstract

High-quality demonstration data is the primary bottleneck in robot learning research. Teleoperation-based collection yields only 10–20 episodes per hour, requires skilled operators, and suffers from quality variance due to operator fatigue and proficiency differences.

We present RAPIDS, a multi-agent system that fully automates the robot data collection and learning pipeline. From a single natural language instruction, RAPIDS autonomously converts the task into a structured specification (A-1), generates a simulation environment in IsaacLab (A-2), collects demonstration data via LLM-generated code in both simulation and the real world (A-3), validates quality through dual pre/post verification (A-4), executes on heterogeneous robots via a unified control framework (B-1), and fine-tunes VLA models with LoRA adapters (B-2).

RAPIDS produced 57 GB of simulation data across 13 tasks (78% episode success rate) and 10 GB of real-world data across 14 tasks on 3 robot platforms (SO-101 single-arm, SO-101 dual-arm, UR7e), reducing new-task pipeline time from 4–8 weeks to 17 minutes and per-episode cost from $10–50 (operator labor) to ~$0.02 (API cost).

System Overview

RAPIDS consists of 6 specialized agents orchestrated in an end-to-end pipeline. The system operates across two tracks: Simulation (A-1 → A-2 → A-3 → A-4) for scalable data production in IsaacLab, and Real-World (A-3 → B-1 → B-2) for physical robot execution and VLA model deployment.

System Architecture Diagram PLACEHOLDER: 6-Agent pipeline overview with Sim/Real-World tracks

Pipeline Flow: A natural language command (e.g., "Stack the blocks inside the tray") is processed by A-1 into a YAML task spec, A-2 generates an IsaacLab simulation environment, A-3 collects demonstrations via LLM-generated code (in both sim and real), A-4 validates code and data quality through dual verification, B-1 provides unified heterogeneous robot control, and B-2 fine-tunes VLA models with LoRA adapters — all without human intervention.

Core Algorithms

LLM-based IsaacLab Code Generation

A 679-line system prompt (prompts/isaaclab_generation.md) defines YAML → IsaacLab ManagerBasedRLEnv code conversion rules, with category-specific reference code as few-shot examples. On execution failure, the error log is fed back to the LLM for up to 5 automatic self-refinements. For complex assembly tasks, deterministic templates bypass LLM hallucination.

Code-as-Policies Data Collection

The LLM generates skill sequences by composing Perception APIs and action primitives. Execution uses Pinocchio (primary) + IsaacLab DifferentialIK (fallback) for 6-DOF IK (palm-down default → ±15°/±30° tilt fallback → 3-DOF fallback, num_random_samples=30). A 3-tier success validation (geometric check → VLM Judge → environment flag) filters out failed episodes immediately.

Dual Pre/Post Verification

Pre-execution: AST-based static analysis checks whether LLM-generated code satisfies the YAML spec, returning concrete violation feedback for iterative correction. Post-execution: VLM receives rendered screenshots (front/top views) and scores environment creation success and task completion quality on a 100-point scale (Scene Fidelity 40 + MDP Correctness 20 + Task Alignment 15 + Runtime Validity 25).

LoRA Adapter VLA Fine-Tuning

LeRobot v3.0 demonstration data is preprocessed (80/20 train/val split, normalization statistics) to fine-tune pre-trained VLA models (GR00T N1.5, Pi0.5) via LoRA adapters. Task-specific adapters are hot-swapped in < 10ms through the B-1 Controller module.

Key Differentiators

 End-to-End Autonomy

From a single natural language input, the Orchestrator coordinates all 6 agents to produce trained VLA models — no manual environment setup, data collection, or quality review required. Reduces pipeline time from 4–8 weeks to 17 minutes.

 Closed-Loop Quality

Dual pre/post verification with automatic feedback loops: A-2 self-refinement (up to 5 retries), A-3 3-tier success validation (geometric → VLM Judge → environment flag), A-4 AST static analysis + VLM visual scoring, and B-1 real-time 10Hz monitoring.

 Physical World Design

Physics-aware at every level: Pinocchio 6-DOF IK with reachability verification, PhysX 5 stability checks, automatic Domain Randomization for sim-to-real transfer, and low-level safety limits (Trajectory/Joint/Cartesian + gripper control).

Agents

Each agent is a specialized module with its own AI models, inputs/outputs, and verification mechanisms. Together they form a fully automated pipeline spanning task specification to model deployment.

Task Definition Agent [A-1]

Converts natural language task descriptions into structured YAML specifications via RAG-powered LLM pipeline with a library of 82 indexed tasks.

GPT-5 GPT-5-mini all-MiniLM-L6-v2 RAG
Input: Natural language task description
Output: Structured YAML task specification
  • LLM-based NL parsing and task decomposition
  • Vector search RAG (82 task library indexed)
  • Structured YAML spec generation with constraints

Simulation Generation Agent [A-2]

GPT-5 GPT-5-mini Gemini IsaacLab
Input: YAML task specification
Output: IsaacLab ManagerBasedRLEnv Python code
  • LLM code generation with 679-line system prompt
  • Self-refinement loop (up to 5 auto-corrections)
  • Adaptive Domain Randomization (YAML → EventTermCfg)
  • VLM-based visual environment verification
PLACEHOLDER: A-2 simulation screenshot
Result: Code Runnability 87/100 • 4 robot types • 8 task categories

Data Collection Agent [A-3]

Collects robot demonstration data in both simulation (IsaacLab) and real-world environments using LLM-generated control code with automatic Forward-Reset loops.

Gemini 3.1 Flash GPT-5.2 Pinocchio IK LeRobot v3.0
Input: Task instruction + environment/scene + primitive skills
Output: LeRobot v3.0 demonstration dataset
  • 6-DOF IK (Pinocchio + DifferentialIK fallback)
  • 3-tier success validation (Geometric → VLM → Env flag)
  • Forward-Reset loop for continuous unattended collection
  • Bi-arm coordination + multi-camera recording
  • Random seed placement for data diversity

Pre & Post Validator [A-4]

Dual verification system: pre-execution static code analysis (AST-based) and post-execution visual validation (VLM-based) with 100-point scoring.

GPT-5 GPT-5-mini AST Parser VLM Evaluator
Input: LLM-generated code + YAML task spec
Output: Score (0–100) + violation feedback
  • Scene Fidelity: asset completeness, physics, robot config
  • MDP Correctness: observation/reward/termination validity
  • Task Alignment: goal condition mapping, threshold check
  • Runtime Validity: env creation, physics stability, NaN/Inf
  • VLM visual scoring on front/top screenshots
PLACEHOLDER: A-4 verification pipeline image
Result: Static + visual dual verification with auto-correction feedback loop

Robot Control Framework [B-1]

Hierarchical heterogeneous robot control pipeline with ROS2 communication, plugin-based robot adapters, and real-time VLM monitoring.

GR00T N1.5 OpenVLA LeRobot ROS2 Humble LangChain
Input: NL command + RGB/RGBD image + model selection
Output: Robot trajectory / joint control commands
  • 5-stage pipeline: Perception → Planner → Controller → Robot → Monitor
  • Plugin adapter system (CustomRobot/RobotBackend abstraction)
  • Trajectory/Joint/Cartesian command types + gripper control
  • Real-time 10Hz monitoring with VLM visual judgment
PLACEHOLDER: B-1 multi-robot control demo
Result: RoboCasa 16 tasks + LIBERO 10 tasks • Franka & UR7e validated

VLA Fine-Tuning [B-2]

Fine-tunes pre-trained VLA models with LoRA adapters using demonstration data collected by A-3, enabling task-specific adapter switching in under 10ms.

GR00T N1.5 SmolVLA Pi0.5 LoRA
Input: A-3 demonstration data + pre-trained VLA model
Output: Fine-tuned VLA model with task-specific LoRA adapter
  • LoRA adapter-based VLA fine-tuning pipeline
  • LeRobot v3.0 data preprocessing (80/20 train/val split)
  • Task-specific adapter hot-swapping (< 10ms)
  • 3 VLA backend support (GR00T, SmolVLA, Pi0.5)

Real-World Collection Detail [A-3]

Real-world data was collected autonomously on three robot configurations: SO-101 single-arm (7 tasks, 3.1 GB), SO-101 dual-arm (3 tasks, 5.1 GB), and UR7e (4 tasks, 1.4 GB). The Forward-Reset loop enabled continuous collection without human intervention, while random seed placement within workspace constraints ensured data diversity. Task categories span grasping, arranging, stacking, insertion, assembly, and deformable object manipulation.

Comparison with Conventional Approaches

RAPIDS replaces manual, fragmented workflows with an integrated autonomous pipeline. The following table summarizes the key differences across each stage of the robot learning lifecycle.

Aspect Conventional RAPIDS
Task Definition Manual reward/env code — days to weeks per task [A-1] NL → RAG + LLM → YAML in minutes
Sim Environment Hand-authored env code, manual Domain Randomization [A-2] Auto-generated IsaacLab code, self-refinement (5 retries), auto DR, VLM visual check
Data Collection Teleoperation: 10–20 ep/hr, requires expert operator [A-3] Code-based autonomous collection, 6-DOF IK + 3-tier validation. Sim: 13 tasks (57 GB), Real: 14 tasks (10 GB)
Quality Assurance Post-hoc manual review or none — errors propagate [A-4] Pre (AST static analysis) + Post (VLM visual scoring) dual verification with auto-correction loop
Robot Control Per-robot bespoke code, full rewrite on platform change [B-1] 5-stage hierarchical pipeline, ROS2 plugin adapters, LoRA adapter swap < 10ms
VLA Fine-Tuning Manual data curation and training setup [B-2] LoRA adapter fine-tuning pipeline with LeRobot v3.0 preprocessing
Extensibility New task/robot requires full pipeline rewrite Add a YAML spec or robot_profiles/*.yaml — LLM/VLM multi-backend swappable

Impact

 Research Productivity

Automates the biggest bottleneck in robot learning — data collection — reducing experiment iteration cycles from weeks to days. RAPIDS produced 13 simulation tasks (57 GB) and 14 real-world tasks (10 GB) in batch mode without human intervention, accelerating Embodied AI research at large.

 Lower Barrier to Entry

Previously, environment setup alone took 1–2 weeks and data collection 2–4 weeks, totaling 4–8 weeks of engineering effort. RAPIDS compresses this to 5–7 days with natural-language-level task definitions. 82 task YAMLs, 4 robot profiles, and a plugin architecture enable instant extension via config files.

 Open Data Ecosystem

All demonstration data is published in LeRobot v3.0 (Open X-Embodiment compatible) format on HuggingFace. Source code, models, and data are openly available, addressing the data scarcity problem in VLA model research and improving reproducibility across the community.

 Industrial Applicability

Conventional teleoperation costs $10–50 per episode (operator labor). RAPIDS replaces this with simulation compute at ~$0.02 per episode (API cost). Multi-robot support enables multi-platform deployment in manufacturing, logistics, and service robotics with dramatically reduced data collection and model training costs.
